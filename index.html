  2. export them
  3. import them in other files

big picture

Observables 'pipe' data from a source to its subscriber(s)

They can transform those values/objects along the way as well

Creating an observable

Observable.of(val)     

     from a static value

 

Observable.from(obs | collection)     

     from an iterable collection, another observable, etc.

These are observables: 

                            let obs1 = Observable.of(1, 2, 3);
let obs2 = this._http
    .get('http://domain.com/api/profile/43242');
let obs3 = Observable.from(obs2);

So that means I can start getting data, right? 

                            let result = this._http
    .get('http://domain.com/api/profile/43242');

console.log(result);

Nope!

Observables are "cold" by default 

                            this._http
    .get('http://domain.com/api/profile/43242')
    .subscribe(result => console.log(result));

Call subscribe to make an observable "hot"

Every time you call subscribe(),

the producer gets recreated 

                                // bad code

getProfileData(id: number) {
    let obs = this._http
        .get(`http://domain.com/api/profile/${id}`);
    
    obs.subscribe(result => console.log(result));

    return obs;
}

......

this._profiles.getProfileData()

The HTTP call is made twice in this example

You can share the same producer for multiple subscriptions: 

                                let obs = Observable.interval(500).share();

let sub1 
    = obs.subscribe(result => console.log(result));

let sub2
    = obs.subscribe(result => console.log(result));

The observable immediately becomes "hot" upon calling share()
